Hash a cat?

In this section, it's about defining the hake. A transformation function transformed into another possibility to find the starting data. It's not about encrypting the data. The principle of data transformation is fast, unique for each input.

In fact, creating a perfect hash function is impossible: most of the time, hash functions can cause collisions. Modern hash functions use operations such as additions, rotations, or Boolean operations such as XOR, OR, or NOT. These operations are performed on the bits of the data to be encrypted, and therefore easy and quick to apply. The most popular hash functions are:


  • md5

    • It works in a rather simple scheme, first, we add at the beginning of the data to be encrypted to have a length multiple of a number (512 bits), then we apply the same process to each block of 512 bits of length, which is divided into sub-blocks of 32 bytes. We apply functions to each of these blocks, we add them, concatenate them, and here is our output data.

    Although this function is quick to use, it is very complicated to obtain input data using brute force; it is the principle of Kerckhoffs. However, since this function was created a while ago, there are many sites to find the possible texts used to generate a hash. This is one of the limitations of this method. It is therefore today less and less used.


  • sha-1

    • The sha-1 function is considered the second most used, but it was recently discouraged because of its low robustness against large resources.


  • sha-256

    • Is the sha-256 function considered robust and efficient and is probably the favorite today.


We intend to use these three functions because they are the most used today in password databases. There are hundreds of others but they are only used in specific cases (password managers, windows / linux passwords etc.) that we do not intend to support in priority.
