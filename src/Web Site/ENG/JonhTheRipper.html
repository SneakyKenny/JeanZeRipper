John what?

John The Ripper also known as JTR, can crack a password and has three modes for breaking: simple mode, dictionary attack and incremental attack.

In addition to JTR, Hashcat allows you to use the following techniques: Mask Attack, Hybrid Attack, Rule Attack, and Swap Attack. Some of these techniques are described below.

We will now define the different methods of breaking. There are several methods of breaking passwords and each has its advantages and disadvantages, we will focus primarily on these methods:


  • Brute force:

    • The brute force method is the simplest. It makes it possible to generate all the combinational possibilities and to test them one by one. It has a success rate of 100%. However, it is very expensive in terms of time depending on the speed of the machine used, it can take up to years to crack a password.


  • Dictionary:

    • The dictionary method makes it possible to take as argument a text file containing a list of words in order to test them one by one. This method therefore does not allow to find a password with certainty but can, however, be quick to perform compared to that of brute force.


  • Mask:

    • The method by mask, is a method which makes it possible to reduce considerably the possible combinations and therefore the time of breaking compared to the brute force, by adding conditions. Nevertheless, it requires to know information in advance on the password as the size of the password, the position of capital letters and so on.


  • Combinaison :

    • The combination method is a rather effective method but requires to know beforehand information about the user such as his date of birth, name, first name, hobbies etc., which are put in a dictionary and combined to find the password example: "louis29021999villejuifepitafootball".


  • Rainbow table:

    • This method consists of chopping or retrieving the hash of as many words as possible and associating them with the plaintext in a database. When the attacker would like to find a password, it will be enough to look for the hash in the database and to recover the clear text, if it is there. This technique is extremely effective for simple or widely used passwords because there is no calculation, just do the research. The disadvantage is in the storage of the database, which is often tens of Giga Bytes. Because of the storage inconvenience, we do not intend to implement this method. To gain in efficiency, the mixture of several of these methods is possible.
