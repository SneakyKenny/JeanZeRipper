Explanations:

With parallelism, especially multithreading, we can distribute the workload across multiple cores or processors. This greatly divides the execution time of some programs:

If you can use four processors at the same time instead of just one, you can essentially divide the execution time by four. This is very effective for recurring calculations over long periods of time, and thus breaking passwords (also referred to as "Embarrassingly parallel problem").

Nevertheless, multithreading requires a particular code layout from the beginning.

There are several libraries for multithreading. We will focus on OpenMP and POSIX threads. OpenMP is quite easy to use, since it makes it easy to use POSIX threads. We will use it at the beginning to have a functional code, then we will try to use only POSIX threads to learn more about multithreading. There are probably other libraries for parallelism, but in this way we can use existing libraries.

The simplicity of OpenMP allows in a few lines for four cores to make a quarter of the iterations of a for-heart loop, whereas without OpenMP it is necessary, for example, first to look for the number of processors in order to create the number of adequate threads, then manually create the threads and manipulate the data more difficult.
